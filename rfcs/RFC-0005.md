1. Проблема
-----------

Информация, необходимая для принятия управленческих решений, распределена по множеству инструментов: транскрипции встреч (Circleback), переписка в Slack и Telegram, документы и файлы. Каждый инструмент — отдельный силос с собственным поиском, который не знает о существовании остальных.

На практике это означает:

* **Ни у кого нет целостной картины.** Объём информации в чатах, каналах и встречах превышает возможности одного человека. Никто не в силах читать все каналы в Slack и Telegram и присутствовать на всех встречах — а значит, каждый руководитель видит только фрагмент происходящего. Решения принимаются на неполных данных.
* **Контекст решений теряется.** Обсуждение начинается в Slack, продолжается на встрече (Circleback), итоги фиксируются (или не фиксируются) в документе. Восстановить полную картину — ручная работа на десятки минут.
* **Институциональное знание уходит с людьми.** При текучке кадров контекст принятых решений, договорённостей и причин выбора теряется, потому что он существует только в головах участников и разрозненных чатах.
* **Поиск неэффективен.** Чтобы найти, что решили по конкретному вопросу, нужно искать в нескольких системах параллельно, знать, где именно было обсуждение, и вручную сопоставлять фрагменты.

**Цель:** дать топ-менеджменту и их персональным AI-ассистентам доступ к агрегированному знанию компании, чтобы они могли отвечать на вопросы с полным контекстом из всех источников.

2. Предлагаемое решение
-----------------------

Развернуть **Ontology Service** — проприетарный продукт компании Codos для агрегации корпоративных знаний — на выделенной виртуальной машине в AWS-инфраструктуре Fortis.

Ontology Service доступен как в варианте on-prem (развёртывание в инфраструктуре клиента), так и в варианте облачного хостинга (на серверах Codos). Fortis принял решение разворачивать on-prem — данные остаются в инфраструктуре компании.

### Архитектура

Сервис состоит из трёх слоёв:

**Ingestion (сбор данных).** Автоматическая интеграция с источниками данных:

* **Circleback** — транскрипции встреч (через API).
* **Slack** — сообщения из публичных каналов (через Slack API).
* **Telegram** — сообщения из публичных каналов (через Telegram Bot API).
* **Ручная загрузка** — произвольные документы и файлы через веб-интерфейс.

Из Slack собираются **только публичные каналы**. Из Telegram собираются только каналы **куда добавлен бот**.

**Processing (обработка).** Собранные данные обрабатываются с использованием Claude API (Anthropic): структурирование, извлечение ключевых фактов, решений и договорённостей, построение связей между фрагментами из разных источников.

**Interface (доступ).** Два способа взаимодействия с агрегированной базой знаний:

* **Веб-интерфейс** — чат на сайте сервиса, где можно задавать вопросы на естественном языке и получать ответы с цитатами и ссылками на источники.
* **MCP для персональных AI-ассистентов** — подключение через MCP из OpenClaw, NemoClaw, Codos и других инструментов. Онтология — контекст компании (публичные каналы); личный контекст (закрытые чаты, почта, календарь) остаётся у персонального ассистента, который обращается к онтологии как к внешнему источнику.

### Авторизация

Google OAuth с белым списком email-адресов. Доступ только у явно одобренных сотрудников. Разграничения внутри онтологии нет — все авторизованные пользователи видят все данные.

### Инфраструктура

| Параметр | Значение |
| --- | --- |
| Размещение | On-prem: AWS, выделенная VM в инфраструктуре Fortis |
| Ресурсы | 4 vCPU, 16 GB RAM |
| Развёртывание | Codos |
| Текущее обслуживание | Codos (мониторинг, поддержка, обновления) |
| Стоимость Claude API | До $1 000/мес |

### Целевая аудитория

* **Первая версия (текущее предложение):** топ-менеджмент компании. Полный набор источников данных, включая управленческие встречи и стратегические обсуждения.
* **В перспективе:** вторая версия для широкого круга сотрудников с ограниченным набором данных (без конфиденциальных управленческих обсуждений).

3. Почему не альтернативы
-------------------------

### Выбор продукта

Ontology Service выбран топ-менеджментом компании как инструмент, наиболее соответствующий задачам управленческой команды.

### Почему on-prem, а не облачный хостинг

Ontology Service доступен в двух вариантах развёртывания: on-prem (в инфраструктуре клиента) и облачный (на серверах Codos). Облачный вариант отвергнут по требованиям ИБ: сервис агрегирует конфиденциальную управленческую информацию (транскрипции стратегических встреч, внутренние обсуждения), и хранение этих данных за пределами инфраструктуры компании неприемлемо. При on-prem развёртывании сырые данные и агрегированная база остаются на VM Fortis.

4. Что теряем (компромиссы и техдолг)
-------------------------------------

### Риски

| Риск | Severity | Митигация |
| --- | --- | --- |
| **Данные передаются в Claude API для обработки** | Высокий | Anthropic: SOC 2 Type II, ISO 27001, не использует данные бизнес-клиентов для обучения. Передаются фрагменты для обработки, не полный дамп. Сырые данные и агрегированная база хранятся на VM Fortis, не у Anthropic. |
| **Доступ к конфиденциальной управленческой информации** | Высокий | Google OAuth + белый список. Доступ только у явно одобренных сотрудников. Аудит-лог обращений. |
| **Зависимость от Codos для обновлений** | Средний | По контракту, при прекращении сотрудничества Codos передаёт Fortis полную копию исходного кода. 30 дней переходного периода с поддержкой и передачей знаний. Fortis вправе использовать и дорабатывать продукт самостоятельно. Сервис продолжает работать без обновлений. |
| **Доступность Claude API** | Низкий | При недоступности API новые данные не обрабатываются, но уже агрегированная база остаётся доступной для поиска и вопросов (по локальному индексу). |

### Техдолг

Сервис разворачивается на отдельной VM и не затрагивает существующую инфраструктуру и кодовую базу. Техдолг в существующих системах не создаётся. При отказе от сервиса — удаление VM.

### Информационная безопасность

| Параметр | Значение |
| --- | --- |
| Где хранятся данные | On-prem: AWS VM в инфраструктуре Fortis |
| Что передаётся в Anthropic | Фрагменты данных для AI-обработки (не полный дамп) |
| Сертификации Anthropic | SOC 2 Type II, ISO 27001:2022 — trust.anthropic.com |
| Обучение на данных | Нет — privacy.claude.com |
| Источники: Slack, Telegram | Из Slack собираются только публичные каналы (выбор каналов определяет администратор Slack при установке приложения). Из Telegram собираются только каналы, куда добавлен бот (добавляют администраторы каналов). |
| Источник: Circleback | Транскрипции встреч, уже хранящиеся в облаке Circleback |
| Авторизация | Google OAuth + белый список email-адресов |
| Шифрование | TLS in transit; шифрование диска VM at rest (AWS EBS encryption) |

5. Влияние на другие команды / юниты
------------------------------------

| Кто затронут | Как именно | Согласование |
| --- | --- | --- |
| **Топ-менеджмент** | Получают доступ к сервису. Онбординг ~10 минут (авторизация + знакомство с интерфейсом). | В процессе согласования |
| **ИБ** | Данные компании (транскрипции, сообщения из публичных каналов) обрабатываются через Claude API. Сырые данные хранятся на AWS VM Fortis. | Требуется согласование Толстых |
| **DevOps / инфраструктура** | Выделение VM в AWS (4 vCPU, 16 GB RAM). Текущее обслуживание (мониторинг, перезапуск). | Требуется согласование |
| **Администраторы Slack** | Предоставление доступа Slack App к публичным каналам. | Требуется согласование |
| **Другие команды** | Не затронуты. Сервис не взаимодействует с существующими системами (GitLab, CI/CD, продуктовый код). | — |

Решение, запрашиваемое у архкома
--------------------------------

Утвердить развёртывание Ontology Service на AWS-инфраструктуре Fortis. Конкретно:

1. **Утвердить размещение сервиса** на выделенной VM в AWS Fortis (4 vCPU, 16 GB RAM).
2. **Разрешить подключение источников данных:** Circleback (API), Slack (публичные каналы), Telegram (публичные каналы).
3. **Утвердить использование Claude API** (Anthropic) для обработки корпоративных данных.
4. **Утвердить доступ** для белого списка сотрудников топ-менеджмента через Google OAuth.
