Decision
========

**В контексте** масштабирования AI code review на все инженерные команды Fortis (9+ репозиториев, стек .NET 9 / Angular 20 / Python-FastAPI / Kotlin, compliance-периметр UAE),

**сталкиваясь с** невозможностью кастомизации пайплайна ревью в SaaS-инструментах (Greptile, CodeRabbit, Sourcery, GitLab Duo) -- отсутствие доменных проверок, обучения на реакциях команды, интеграции с внутренними источниками контекста (БЗ, API-контракты, Kaiten) и гранулярного контроля расходов,

**мы решили** создать собственный сервис `fortis-mr-reviewer` на базе Claude Agent SDK, развернутый в Yandex Cloud Managed Kubernetes как FastAPI-приложение с agentic-архитектурой (worktree + tool use + structured output),

**и отказались от** Greptile SaaS, Greptile self-hosted, CodeRabbit, Sourcery, GitLab Duo и развитие существующего самописного скрипта,

**чтобы достичь** полный контроль над пайплайном ревью и работой агента в изолированном worktree с полным доступом к коду (а не по статическому индексу или diff), обучаемости через Consolidator с git-аудитом, сделав задел на интеграцию с доменным контекстом и гранулярный контроль расходов per-MR /per-repo / per-role,

**принимая** зависимость от Anthropic API как осознанный vendor lock-in уровня LLM-провайдера, необходимость owner'а и дежурства для поддержки сервиса, а также передачу запрошенных агентом сниппетов кода во внешний API (с аудитом и DPA/SOC 2 Type II у провайдера).
