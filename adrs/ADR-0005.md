# ADR-0005: LLM Gateway MVP на внешнем VPS

> **Источник:** [Confluence](https://bz.incdb.team/pages/viewpage.action?pageId=3417997325)

**Статус: Accepted**

---

Контекст
========

Сотрудники Life-Pay используют ChatGPT и Claude через личные аккаунты для рабочих задач. Персональные данные клиентов (ФИО, номера карт, паспортные данные) уходят во внешние API без контроля компании. Нет учёта расходов по пользователям, нет единой точки входа.

---

Решение
=======

Развернуть корпоративный LLM Gateway как набор Docker-контейнеров на внешнем VPS (4 vCPU, 8 ГБ RAM, 80 ГБ SSD) для проведения пилота на 100-200 сотрудников.

Стек: Nginx (reverse proxy, TLS, rate limiting) -> Open WebUI (веб-интерфейс) -> PII Masker (маскирование ПД через NER/SpaCy) -> LiteLLM (агрегатор API, бюджеты) -> PostgreSQL. Исходящие запросы к Anthropic и OpenAI идут через HTTP proxy (AWS) под контролем компании.

Целевое решение после пилота — переезд в k8s Life-Pay: Ingress вместо Nginx, managed PostgreSQL, SSO через корпоративный IdP, Grafana-on-call вместо watchdog, корпоративный VPN вместо HTTP proxy.

---

Последствия
===========

Плюсы:

* ПД не передаются во внешние API в исходном виде — маскируются до отправки
* Централизованный учёт расходов и бюджетные лимиты на пользователя ($5/мес) и суммарно ($1000/мес)
* Единая точка входа для всех сотрудников, единые API-ключи под контролем компании
* Сервис поднят и работает, пилот можно начать немедленно

Минусы:

* История диалогов с оригинальным текстом хранится в PostgreSQL на внешнем VPS вне инфры LP — риск по 152-ФЗ, закрывается переездом в k8s
* PII Masker не даёт 100% покрытия — нестандартные формулировки могут быть пропущены
* Загрузка файлов и изображений заблокирована (маскирование изображений не реализовано)
* Нет HA — единый PostgreSQL, восстановление при падении ~1 час, потеря данных до 24 часов
* Авторизация email/пароль, ручное управление учётками — закрывается SSO при переезде в k8s
